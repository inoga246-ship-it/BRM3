# BRM Map（地図モード・別プロジェクト）

BRM PACE MANAGER（`BRM-main`）から呼び出される、地図表示専用の別プロジェクトです。
既存のペース管理アプリには一切変更を加えず、完全に独立したファイル構成にしています。

## 構成

```
BRM-map/
├── index.html      地図画面の本体
├── css/style.css   見た目（BRM-mainと同じダーク×ネオン系トーン）
├── js/script.js    地図表示・GPS追従・オフラインタイル管理
└── sw.js           アプリ本体（HTML/CSS/JS）をオフラインキャッシュするService Worker
```

## 主な機能

- **Leaflet.js + OpenStreetMapタイル**による地図表示
- **現在地への常時追従**（`watchPosition`）。このページを開いている間だけGPSを常時監視し、ページを離れると自動で停止します（電池消耗を抑えるため）。自分の移動と共に地図も常にスクロール（追従）し、手動で地図をドラッグすると追従が一時解除されます（🎯アイコンで現在地表示に復帰）
- **画面の自動消灯防止**（Wake Lock API）。☰メニュー内の「画面を常にONにする」トグルでON/OFFを切り替え可能（デフォルトON）。バックグラウンド復帰時は希望状態に応じて自動的に再取得します
- **地図の向き切替**（地図上の🧭フローティングアイコン）：「北が上」（地図は固定、矢印が実際の進行方向を指す）と「進行方向が上」（地図そのものが回転し、矢印は常に上を指す）を切り替え可能。進行方向はGPSの移動方向（`coords.heading`）から取得するため、停止中は方角を取得できず直前の向きを維持します。「進行方向が上」モード中は地図のドラッグ操作を無効化しています（回転中はドラッグ方向と画面表示がずれるため）。
- **ルートの進行方向矢印**：ルート全体に対して間隔を空けて（最大40本程度）控えめな半透明の矢印を表示し、進行方向を視覚的に把握しやすくしています（邪魔にならない密度・透過度に調整済み）
- **往復ルート対策**：直前にマッチした距離の±8km以内を優先的に探索する連続性チェックにより、同じ道を2度通る区間でも往路/復路の取り違えを抑制
- **PC（チェックポイント）・休憩スポットのマーカー表示**：BRM PACE MANAGER側の①②リストを読み取り、ルート上の該当距離に自動でマーカーを配置（タップで地点名・距離をポップアップ表示）
- **現在地マークの形状選択**：☰メニューから「矢印／丸／🚲自転車／📍ピン」の4種類＋色を選択可能
- **地図上アイコンのフリー移動**：ズームボタン（＋－セット）・🧭向き切替・🎯現在地復帰の3つは、いずれも長押し（約450ms）でドラッグモードに切り替わり、画面上の好きな位置に移動できます。位置は端末に保存され、☰メニューの「地図上アイコンの位置を初期化」でいつでもデフォルト位置に戻せます
- **タイルのオフラインキャッシュ**：表示したタイル画像をCache Storage APIで自動保存し、次回以降は通信せずに再利用します
- **メニュー内「💾 エリア保存」**：誤操作防止のため☰メニューの中に配置。Wi-Fi等の通信環境がある状態で押すと、ルート沿いのタイル（ズームレベル13〜16、隣接タイルも含む）を4枚並列でダウンロードして保存できます
- **ルート逸脱時の音通知**：ルートから300m以上離れるとビープ音で通知（逸脱継続中は30秒間隔で再通知）。☰メニューでON/OFF切替可能

## BRM-mainとの連携（同一オリジン配置時）

`BRM-main`と同じサーバー・同じドメイン配下に配置すると、`localStorage`を介して以下のデータを共有します。

- 起動時：`gpxTrackPoints`（GPXの解析済みルートデータ）、`pcList3`・`shopList3`（①②リスト）を読み込み、自動的にルート線・各マーカーを描画
- GPS追従中：マッチした現在距離を`distance`キーへ書き込み、`BRM-main`に戻った時にも反映されるようにする

GPXファイル自体は地図側では読み込みません（BRM-main側で読み込んだものをそのまま共有して使う設計です）。別ドメインで運用する場合はこの自動連携ができないため、その場合は別途連携方法（JSONバックアップ機能の共有など）を検討する必要があります。

## 戻るボタンについて

「← 戻る」は基本的にブラウザの**履歴で戻る**（`history.back()`）動作です。BRM-mainの🗺️ボタンから遷移してきた場合は、配置場所によらず正しく戻れます。直接このURLを開いた場合（履歴が無い場合）のみ、同階層の`index.html`へ遷移するフォールバックになっています。

## 完全オフライン起動について

**Leaflet本体（CSS/JS/マーカー画像）はこのプロジェクト内にローカル同梱（vendor化）する構成になっています。** `js/vendor/leaflet/` 配下に以下の5ファイルを配置してください（このリポジトリには同梱されていないため、初回のみ手動でダウンロードが必要です）。

```
js/vendor/leaflet/leaflet.js
js/vendor/leaflet/leaflet.css
js/vendor/leaflet/images/marker-icon.png
js/vendor/leaflet/images/marker-icon-2x.png
js/vendor/leaflet/images/marker-shadow.png
```

ダウンロード例（PCのターミナルで実行、`BRM-map`フォルダ直下で）：

```
mkdir -p js/vendor/leaflet/images
curl -o js/vendor/leaflet/leaflet.js https://unpkg.com/leaflet@1.9.4/dist/leaflet.js
curl -o js/vendor/leaflet/leaflet.css https://unpkg.com/leaflet@1.9.4/dist/leaflet.css
curl -o js/vendor/leaflet/images/marker-icon.png https://unpkg.com/leaflet@1.9.4/dist/images/marker-icon.png
curl -o js/vendor/leaflet/images/marker-icon-2x.png https://unpkg.com/leaflet@1.9.4/dist/images/marker-icon-2x.png
curl -o js/vendor/leaflet/images/marker-shadow.png https://unpkg.com/leaflet@1.9.4/dist/images/marker-shadow.png
```

配置後、Service Worker（`sw.js`）がこれらもアプリ本体と一緒にオフラインキャッシュするため、**2回目以降の起動はLeaflet本体も含めて完全にネット接続不要**になります（1回目のアクセス時にService Workerのインストールが必要なため、最初の1回だけはネット接続がある状態でアプリを開いてください）。

配置を忘れた場合でもアプリ自体は起動しますが、Leafletの読み込みに失敗し地図が表示されません（ブラウザのコンソールに404エラーが出ます）。

### オフライン時の挙動

- 画面上部に **「オフライン」バッジ** が表示され、現在オフラインで動作していることがわかります（`navigator.onLine`のオンライン/オフライン切り替えイベントで自動更新）。
- ルート周辺タイル未保存の範囲でオフラインになった場合、タイル画像は壊れたアイコンではなく目立たない灰色プレースホルダーで表示されます（エラーが連続で発生してアプリが重くなることも防止）。
- GPS位置取得自体は端末のGPSハードウェアを使うため、通信が無くても動作します。

## 既知の制限・今後の課題

- 「進行方向が上」モードの地図回転は、CSSのtransformでビジュアルを回転させる簡易的な実装です。回転中のピンチズーム操作は見た目と実際の中心がずれる場合があります（フォロー中心固定の自動追従との併用を推奨）。
- タイルの事前ダウンロードはズームレベル13〜16のみを対象にしています。より詳細な地図（建物形状などが見えるレベル）が必要な場合はズームレベルの追加が必要ですが、その分タイル数・容量が増えます。
- iOS Safari等、Wake Lock APIに対応していない/制限がある環境では、画面の自動消灯防止が機能しない場合があります。

## 開発・確認方法

ローカルで動作確認する場合は、`BRM-map`フォルダ内で簡易サーバーを起動してください（`file://`で直接開くとService WorkerやCache Storage APIが正しく動作しません）。

```
python3 -m http.server 8000
```

ブラウザで `http://localhost:8000/` を開いてください。
